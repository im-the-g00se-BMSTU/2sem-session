# 10. Распределение памяти. Статическое и динамическое выделение памяти. Понятия «стек» и «куча». Функции для выделения и освобождения памяти, особенности их использования, пример работы. Ошибки при работе с динамической памятью (утечки, памяти, висячий указатель, двойное освобождение).

В программе на C память обычно можно условно разделить на несколько областей:

1. Статическая область памяти - для глобальных и статических переменных.
2. Стек - для локальных переменных и вызовов функций.
3. Куча - для динамически выделенной памяти.
4. Область кода - для машинных инструкций программы.

Статическое выделение памяти - это выделение памяти заранее, до выполнения основной части программы или при входе в функцию. (2 способа: со static и без. Переменная со static живёт всё время работы программы, автоматически же созданная переменная (без static) удаляется при выходе из функции, в которой она была использована)

Стек - это область памяти, которая используется для хранения:

- локальных переменных;
- параметров функций;
- адресов возврата из функций;
- служебной информации при вызове функций.

Стек работает по принципу LIFO - last in, first out, то есть «последним пришёл - первым ушёл».

Динамическое выделение памяти - это выделение памяти во время выполнения программы. Куча - это область оперативной памяти, которая допускает динамическое выделение памяти.

Функции для выделения и освобождения памяти:

1. malloc выделяет блок памяти заданного размера в байтах.void \*malloc(size\_t size); Пример: int\* p = (int\*)malloc(sizeof(int));
2. calloc выделяет память под несколько элементов и заполняет её нулями.<br>void\* calloc(size\_t count, size\_t size); Пример: int\* arr = (int\*)calloc(5, sizeof(int));
3. realloc изменяет размер уже выделенного блока памяти. void\* realloc(void\* ptr, size\_t new\_size); Пример: arr = realloc(arr, 5 \* sizeof(int));
4. free освобождает память, выделенную ранее через malloc, calloc или realloc.

Ошибки:

1. Разыменование пустого указателя.
2. Утечка памяти - это ситуация, когда память была выделена, но не была освобождена.

```c
void func() {
    int *p = malloc(sizeof(int));
    *p = 10;
} // free(p) нет
```

1. Висячий указатель - это указатель, который указывает на память, которая уже освобождена или больше не существует.

```c
int *p = malloc(sizeof(int));
*p = 10;
free(p);
printf("%d\n", *p); // ошибка: память уже освобождена
```

1. Двойное освобождение - это попытка вызвать free два раза для одной и той же области памяти.

```c
int *p = malloc(sizeof(int));
free(p);
free(p); // ошибка
```

1. Использование памяти за пределами массива

```c
int *arr = malloc(3 * sizeof(int));
arr[3] = 10; // ошибка, допустимы только arr[0], arr[1], arr[2]
free(arr);
```

1. Потеря адреса выделенной памяти

```c
int *p = malloc(sizeof(int));
p = malloc(sizeof(int)); // первый адрес потерян
```
