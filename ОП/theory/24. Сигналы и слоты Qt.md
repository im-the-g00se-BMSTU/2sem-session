# 24. Qt. Событийная модель: слоты и сигналы. Автоматическое и ручное связывание. Пример нажатия на кнопку. Напишите слово «слот» семинаристу, первый человек в каждой группе получит «сигнал».

Событийная модель Qt<br>В Qt используется событийная модель, основанная на механизме сигналов и слотов.

Сигнал - это сообщение, которое объект отправляет при возникновении события.

Слот - это функция, которая вызывается в ответ на сигнал.

Сигналы и слоты являются слабо связанными: объект, который отправляет сигнал, не обязан знать, какая функция будет вызвана в ответ.

Кто может использовать сигналы и слоты<br>Сигналы и слоты могут использовать классы, которые наследуются от: QObject

или от его потомков, например: QWidget

Так как виджеты Qt наследуются от QWidget, они могут иметь сигналы и слоты. Для работы сигналов и слотов в классе обычно используется макрос: Q\_OBJECT

## Ручное связывание сигналов и слотов

Ручное связывание выполняется с помощью функции: connect()

```c
connect(отправитель, сигнал, получатель, слот);
```

Пример из лекции - новый способ через указатели:

```c
connect(ui->incrementButton, &QPushButton::clicked, this, &MainWindow::onIncrementClicked);
```

Это означает: когда кнопка incrementButton отправит сигнал clicked, у текущего объекта MainWindow будет вызван слот onIncrementClicked.

Старый способ через макросы:

```c
connect(ui->incrementButton, SIGNAL(clicked()), this, SLOT(onIncrementClicked()));
```

Автоматическое связывание(не из лекции)<br>Автоматическое связывание работает по имени слота:

```c
void on_pushButton_clicked();
```

Если кнопка имеет имя pushButton, то при её нажатии автоматически вызывается слот:

```c
void MainWindow::on_pushButton_clicked() {
   ui->label->setText("Кнопка нажата");
}
```
