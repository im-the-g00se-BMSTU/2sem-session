Хеш-таблица - это структура данных, в которой элементы хранятся в виде пар `ключ -> значение`.

Для доступа к элементу используется ключ. Ключ передаётся в хеш-функцию, которая вычисляет индекс ячейки таблицы: index = h(key) % size, где size - размер хеш-таблицы.

## Коллизии

Коллизия - это ситуация, когда разные ключи получают одинаковый индекс в хеш-таблице. Коллизии возникают потому, что количество возможных ключей обычно намного больше количества ячеек в таблице.

## Способы разрешения коллизий

Основные способы разрешения коллизий:

1. Метод цепочек
2. Открытая адресация
	При открытой адресации все элементы хранятся внутри массива таблицы. Если ячейка занята, ищется другая свободная ячейка.

Основные варианты открытой адресации:

1. линейное зондирование;
2. квадратичное зондирование;
3. двойное хеширование.

## Метод цепочек

Метод цепочек - это способ разрешения коллизий, при котором каждая ячейка хеш-таблицы содержит не один элемент, а список элементов, попавших в эту ячейку. Если несколько ключей дают одинаковый индекс, они добавляются в одну цепочку.

#### Добавление элемента

1. Вычисляется индекс по хеш-функции.
2. Проверяется ячейка таблицы.
3. Если ячейка пустая, создается новая цепочка.
4. Если ячейка уже содержит элементы, новый элемент добавляется в список.

#### Поиск элемента

1. Вычисляется индекс по ключу.
2. Открывается цепочка в этой ячейке.
3. В списке последовательно сравниваются ключи.
4. Если ключ найден, возвращается значение.

#### Удаление элемента

1. Вычисляется индекс по ключу.
2. В соответствующей цепочке ищется элемент с нужным ключом.
3. Если элемент найден, он удаляется из списка.

### Плюсы метода цепочек

1. простая реализация;
2. таблица может хранить несколько элементов с одним индексом;
3. удаление элемента выполняется проще, чем при открытой адресации;
4. таблица может работать даже при большом количестве коллизий.

### Минусы метода цепочек

1. требуется дополнительная память для списков;
2. если много элементов попадает в одну ячейку, поиск замедляется;
3. при плохой хеш-функции одна цепочка может стать очень длинной.

### Сложность операций

В среднем случае при хорошем распределении ключей:

1. поиск - O(1);
2. вставка - O(1);
3. удаление - O(1).

В худшем случае, если все элементы попали в одну цепочку:

1. поиск - O(n);
2. вставка - O(n);
3. удаление - O(n).
