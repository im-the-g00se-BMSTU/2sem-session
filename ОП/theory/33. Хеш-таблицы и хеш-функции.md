# 33. Хеш-таблицы. Коллизии. Хеш-функции. Требования к хеш-функциям.

## Хеш-таблица

Хеш-таблица - это структура данных, в которой элементы хранятся в виде пары:

ключ → значение. То есть каждому ключу соответствует некоторое значение.

Примеры из лекции:

data\_1\["BMSTU"\] = "МГТУ";

data\_2\[3.14\] = "PI";

Главная идея хеш-таблицы - получить что-то похожее на массив, но чтобы обращаться к элементам можно было не только по целому индексу, а по ключу произвольного типа: строке, числу, объекту и так далее.

## Ключ и значение

Ключ используется для индексации значения. Ключ может иметь произвольный тип, но в одной хеш-таблице все ключи должны быть одного типа.

Значение - это данные, которые связаны с ключом.

Например:"BMSTU"(ключ) → "МГТУ"(значение)

## Основные операции хеш-таблицы

У хеш-таблицы есть три основные операции:

1. Добавление пары ключ-значение.
2. Удаление по ключу.
3. Поиск по ключу.

Например, при поиске по ключу "BMSTU" таблица должна вернуть значение "МГТУ".

## Хеширование

Хеширование - это процесс преобразования ключа в некоторый индекс с помощью специальной функции. Эта функция называется хеш-функцией.

ключ → хеш-функция → индекс в таблице

## Хеш-функция

Хеш-функция - это функция, которая принимает ключ и возвращает индекс в массиве хеш-таблицы.

Общий вид: index = h(key)

Но так как размер таблицы ограничен, обычно используют остаток от деления:

index = h(key) % size, где size - размер хеш-таблицы.

## Проблема хеширования

Основная проблема заключается в том, что размерность пространства ключей может быть больше размерности пространства индексов. Проще говоря, возможных ключей очень много, а ячеек в таблице ограниченное количество. Из-за этого разные ключи могут попасть в одну и ту же ячейку.

## Коллизия

Коллизия хеш-функции - это получение одинакового индекса для разных ключей. Ключи разные, но индекс получился один и тот же. Это и есть коллизия. Коллизии неизбежны, если количество возможных ключей больше количества ячеек таблицы.

## Методы борьбы с коллизиями

## 1. Метод цепочек

Метод цепочек заключается в том, что в одной ячейке таблицы хранится указатель на список элементов. То есть если несколько ключей попали в один индекс, они не перезаписывают друг друга, а добавляются в связный список.

Плюс метода: простая реализация.<br>Минус: если много элементов попадает в одну ячейку, поиск становится медленнее, потому что нужно проходить список.

## 2. Открытая адресация

При открытой адресации все элементы хранятся прямо в массиве хеш-таблицы. Если нужная ячейка занята, алгоритм ищет другую свободную ячейку по специальному правилу.

Используется обозначение:

h'(k, i)

где:

- k - ключ;
- i - номер попытки;
- size - размер таблицы;
- h(k) - исходная хеш-функция;
- h'(k, i) - изменённая хеш-функция для поиска новой позиции.

## Линейное зондирование

h'(k, i) = (h(k) + i) % size

Если первая ячейка занята, проверяется следующая, потом следующая и так далее.

Проблема: могут образовываться длинные непрерывные занятые участки, которые каждый раз приходится проходить при коллизии.

## Квадратичное зондирование

При квадратичном зондировании расстояние между проверяемыми ячейками растет не линейно, а быстрее.

В лекции указана идея формулы вида: h'(k, i) = (h(k) + a\*i + b\*i^2) % size

Здесь a и b - некоторые константы, которые подбираются в зависимости от размера таблицы.

Смысл: ячейки проверяются не подряд, а с увеличивающимся шагом, поэтому массив получается более разреженным, чем при линейном зондировании.

## Двойное хеширование

h'(k, i) = (h1(k) + i \* h2(k)) % size

Здесь используются две хеш-функции:

1. h1(k) задает начальную позицию;
2. h2(k) задает шаг смещения.

Это уменьшает вероятность плохого скопления элементов.

При открытой адресации удалённые элементы нужно помечать, но не удалять полностью.

## Требования к хеш-функции

К хеш-функции при использовании в хеш-таблицах предъявляются следующие требования:

1. Одинаковый ключ должен всегда давать одинаковый индекс. Иначе мы не сможем найти элемент после добавления.
2. Быстрое вычисление индекса. Хеш-функция должна работать быстро, потому что она вызывается при каждой операции: вставке, поиске, удалении. Если хеш-функция медленная, то хеш-таблица потеряет свое главное преимущество - быстрый доступ.
3. Равномерное распределение индексов. Чем равномернее распределение, тем меньше коллизий.
4. Индекс не должен выходить за размер таблицы. Если размер таблицы равен size, то индекс должен быть в диапазоне: 0 ... size - 1. Поэтому часто используют операцию: index = h(key) % size

## Сложность операций

| Операция | В среднем | В худшем случае |
| --- | --- | --- |
| Вставка | O(1) | O(n) |
| Удаление | O(1) | O(n) |
| Поиск | O(1) | O(n) |
| Расход памяти | O(n) | O(n) |

В среднем хеш-таблица работает очень быстро - за O(1), то есть почти мгновенно. Но в худшем случае, если возникает много коллизий, операции могут стать O(n), потому что придётся просматривать много элементов
