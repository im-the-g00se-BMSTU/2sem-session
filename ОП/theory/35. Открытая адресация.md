# 35. Хеш-таблицы. Коллизии. Способы разрешения коллизий. Открытая адресация (линейное пробирование, квадратичное пробирование, двойное хеширование).

Хеш-таблица - это структура данных, в которой элементы хранятся в виде пар:

ключ → значение

Для определения позиции элемента используется хеш-функция. Она преобразует ключ в индекс ячейки таблицы: index = h(key) % size, где size - размер хеш-таблицы.

## Коллизия

Коллизия - это ситуация, когда разные ключи получают один и тот же индекс. То есть два разных элемента должны попасть в одну и ту же ячейку таблицы. Коллизии возникают потому, что количество возможных ключей больше, чем количество ячеек в таблице.

## Способы разрешения коллизий

Основные способы разрешения коллизий:

1. Метод цепочек
2. Открытая адресация

Метод цепочек - элементы с одинаковым индексом хранятся в списке, связанном с одной ячейкой.

Открытая адресация - все элементы хранятся прямо в массиве хеш-таблицы. Если ячейка занята, ищется другая свободная ячейка по определённой формуле.

## Открытая адресация

При открытой адресации используется модифицированная хеш-функция: h'(k, i), где:

k - ключ;

i - номер попытки;

h(k) - исходная хеш-функция;

size - размер таблицы.

Если первая позиция занята, увеличивается номер попытки i, и вычисляется новая позиция.

## 1. Линейное пробирование

Линейное пробирование - это способ открытой адресации, при котором при коллизии проверяются следующие ячейки таблицы подряд.

Формула: h'(k, i) = (h(k) + i) % size

Недостаток: могут образовываться непрерывные занятые участки таблицы. При новых коллизиях эти участки приходится проходить, поэтому поиск и вставка замедляются.

## 2. Квадратичное пробирование

Квадратичное пробирование - это способ открытой адресации, при котором расстояние между проверяемыми ячейками увеличивается не линейно, а по квадратичному закону.

Формула: h'(k, i) = (h(k) + a\*i + b\*i²) % size

где a и b - константы, которые подбираются в зависимости от размера таблицы. В отличие от линейного пробирования, ячейки проверяются не подряд. За счёт этого элементы распределяются более разреженно.

## 3. Двойное хеширование

Двойное хеширование - это способ открытой адресации, при котором используются две хеш-функции.

Формула: h'(k, i) = (h1(k) + i \* h2(k)) % size

Здесь:

h1(k) - определяет начальную позицию;

h2(k) - определяет шаг смещения;

i - номер попытки.

Если первая ячейка занята, следующая позиция выбирается не просто подряд, а с шагом, который зависит от второго хеша. Этот способ уменьшает вероятность скопления элементов и обычно распределяет элементы лучше, чем линейное пробирование.

## Особенность удаления при открытой адресации

При открытой адресации удалённые элементы нужно помечать, а не просто удалять.

Почему: если полностью очистить ячейку, то при последующем поиске алгоритм может решить, что дальше элемента нет, хотя он мог быть вставлен дальше из-за коллизии.

## Сложность операций

В среднем случае:

вставка - O(1);

поиск - O(1);

удаление - O(1).

В худшем случае, если много коллизий:

вставка - O(n);

поиск - O(n);

удаление - O(n).
