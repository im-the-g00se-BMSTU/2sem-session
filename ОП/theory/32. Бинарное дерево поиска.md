# 32. Динамические структуры данных. Деревья. Корень, лист, высота вершины, глубина дерева. Бинарное дерево и бинарное дерево поиска. Алгоритмы их обработки.

Динамические структуры данных - это составные структуры данных, которые могут изменять свой размер во время выполнения программы. В отличие от статических структур, их размер заранее жестко не фиксирован. Примеры динамических структур: список, стек, очередь, дерево, граф, хэш-таблица. Обычно такие структуры строятся из узлов. Узел хранит данные и ссылки/указатели на другие узлы.

## Дерево

Дерево - это связный ациклический граф. То есть:

1. между любыми двумя вершинами есть путь;
2. в дереве нет циклов.

Формально граф можно записать как: G = (V, E), где V - множество вершин, а E - множество рёбер. Лес - это набор непересекающихся деревьев.

## Основные понятия дерева

1. Вершина / узел - элемент дерева, в котором хранятся данные.
2. Ребро - связь между двумя узлами.
3. Корень - узел, у которого нет родителя.
4. Лист - узел, у которого нет детей.
5. Братья - узлы, имеющие общего родителя.

## Высота и глубина

1. Высота узла - длина самого длинного пути от этого узла до какого-либо листа.
2. Высота дерева - высота корневого узла.
3. Глубина вершины - длина пути от этой вершины до корня.
4. Глубина дерева - длина самого длинного пути от какого-либо листа до корня.

Важно:

- высота любого листа равна 0;
- глубина корня равна 0.

## Бинарное дерево

Бинарное дерево - это дерево, в котором у каждого узла есть не более двух детей.

Обычно их называют: левый потомок, правый потомок.

Пример структуры узла:

```c
typedef struct Node {
   int data;
   struct Node* left;
   struct Node* right;
} Node;
```

В бинарном дереве можно выполнять следующие операции:

1. добавление узла;
2. удаление узла;
3. поиск узла;
4. обход дерева;
5. поиск родителя;
6. поиск братьев узла.

## Бинарное дерево поиска

Бинарное дерево поиска - это упорядоченное бинарное дерево, в котором для каждого узла выполняется правило:

1. значение левого потомка меньше значения родителя;
2. значение правого потомка больше значения родителя.

Это свойство ускоряет поиск, потому что на каждом шаге можно отбрасывать половину дерева.

## Алгоритмы обработки дерева

## 1. Обход в глубину

Обход в глубину бывает трёх видов.

## Прямой обход - pre-order

Порядок:

1. Посетить корень.
2. Обойти левое поддерево.
3. Обойти правое поддерево.

```c
void preorder(Node* root) {
   if (!root)
       return;
   printf("%d ", root->data);
   preorder(root->left);
   preorder(root->right);
}
```

Используется, например, когда нужно сначала обработать сам узел, а потом его потомков.

## Симметричный обход - in-order

Порядок:

1. Обойти левое поддерево.
2. Посетить корень.
3. Обойти правое поддерево.

```c
void inorder(Node* root) {
   if (!root)
       return;
```

```c
   inorder(root->left);
   printf("%d ", root->data);
   inorder(root->right);
}
```

Для бинарного дерева поиска такой обход выводит элементы в отсортированном порядке.

## Обратный обход - post-order

Порядок:

1. Обойти левое поддерево.
2. Обойти правое поддерево.
3. Посетить корень.

```c
void postorder(Node* root) {
   if (!root)
       return;
   postorder(root->left);
   postorder(root->right);
   printf("%d ", root->data);
}
```

Такой обход удобно использовать, например, при удалении дерева, потому что сначала удаляются потомки, а потом сам узел.

## 2. Обход в ширину

Обход в ширину основан на использовании очереди.

Идея:

1. Поместить корень в очередь.
2. Пока очередь не пуста:
  - достать первый узел;
  - обработать его;
  - добавить в очередь его левого и правого потомка.

```c
void bfs(Node* root) {
   if (!root)
       return;
```

```c
   Queue q;
   push(&q, root);
```

```c
   while (!empty(&q)) {
       Node* current = pop(&q);
```

```c
       printf("%d ", current->data);
```

```c
       if (current->left)
           push(&q, current->left);
```

```c
       if (current->right)
           push(&q, current->right);
   }
}
```

## Алгоритмы для бинарного дерева поиска

## Алгоритм поиска в бинарном дереве поиска(не из лекции):

1. Начинаем с корня.
2. Если дерево пустое - элемент не найден.
3. Если значение равно текущему узлу - элемент найден.
4. Если искомое значение меньше текущего - идём в левое поддерево.
5. Если больше - идём в правое поддерево.

```c
Node* search(Node* root, int x) {
   if (!root || root->data == x)
       return root;
```

```c
   if (x < root->data)
       return search(root->left, x);
   else
       return search(root->right, x);
}
```

## Добавление элемента

Добавление в неупорядоченное бинарное дерево - простая операция, а в упорядоченном дереве сложнее, потому что нужно найти подходящее место.

## Алгоритм вставки в бинарное дерево поиска такой(не из лекции):

1. Если дерево пустое - новый узел становится корнем.
2. Если новое значение меньше текущего узла - идём налево.
3. Если больше - идём направо.
4. Когда дошли до пустого места, вставляем туда новый узел.

```c
Node* insert(Node* root, int x) {
   if (!root) {
       Node* node = malloc(sizeof(Node));
       node->data = x;
       node->left = NULL;
       node->right = NULL;
       return node;
   }
   if (x < root->data)
       root->left = insert(root->left, x);
   else if (x > root->data)
       root->right = insert(root->right, x);
```

```c
   return root;
}
```

## Удаление элемента

Есть три случая удаления:

1. Удаление листа.
2. Удаление узла с одним ребёнком.
3. Удаление узла с двумя детьми.

Если удаляется лист, его можно просто удалить.

Если у узла один ребёнок, то ребёнок становится на место удаляемого узла.

## Сложность операций

Бинарное дерево может деградировать в связный список, если элементы вставляются в отсортированном порядке.

В хорошем случае, если дерево сбалансировано:

1. поиск - O(log n);
2. вставка - O(log n);
3. удаление - O(log n).

В плохом случае, если дерево выродилось в список:

1. поиск - O(n);
2. вставка - O(n);
3. удаление - O(n).

## Сбалансированное бинарное дерево поиска

Сбалансированное бинарное дерево поиска - это бинарное дерево поиска, в котором для всех узлов высота левого и правого поддерева отличается не более чем на единицу.

Преимущество - поиск за O(log n).

Примеры методов балансировки:

1. AVL-дерево;
2. красно-чёрное дерево;
3. splay tree / косое дерево.
